# SpriteCraft 0.35.0

Release date: 2026-03-30

## Summary

SpriteCraft 0.35.0 introduces the first real Phase 8 packaging foundation for Windows. The repo now has a defined portable release shape, a bundle-building script, launcher scripts, standalone `studio` output, and packaged LPC runtime asset support.

## Included in this release

- Windows-first packaging strategy and distribution docs
- portable bundle builder for backend, web runtime, and bundled LPC assets
- standalone Next.js `studio` output for packaged builds
- packaged LPC runtime asset resolution through `SPRITECRAFT_LPC_ROOT`

## Migration notes

If you package SpriteCraft after upgrading to 0.35.0:

1. Rebuild the Windows bundle so it picks up standalone `studio` output and the new packaged asset layout.
2. Provide a Node runtime directory when running `tool/build_windows_portable_bundle.ps1`.
3. Do not rely on a git submodule inside packaged builds anymore. Runtime bundles should use `runtime/assets/lpc-spritesheet-creator`.

If you are developing from the repository:

- no migration is required for normal repo-based workflows
- the default runtime path still uses `./lpc-spritesheet-creator`
- submodule update workflows remain unchanged for development checkouts

## Release files

Portable Windows bundles now default to a versioned output directory:

```text
build/release/windows/SpriteCraft-win-x64-0.35.0
```
