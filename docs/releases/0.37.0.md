# SpriteCraft 0.37.0

## Summary

SpriteCraft 0.37.0 adds a dedicated Settings page to Studio and makes a clearer distinction between backend-owned runtime configuration and frontend-owned user preferences.

This release is focused on the last open Phase 8 settings item:

- runtime path visibility for exports, packages, recovery logs, and LPC assets
- backend status visibility for DB-backed history and Gemini
- local Studio preferences for theme, AI surface visibility, history surface visibility, default export preset, and default naming style

## What changed

### Studio settings surface

- Added a `/settings` route in `studio`
- Added a settings panel for local UI preferences
- Added a runtime panel fed from the Dart bootstrap contract
- Added Settings to the primary Studio navigation

### Runtime contract

- Extended `GET /api/bootstrap` with a typed `runtime` block
- Exposed:
  - export directory
  - project package directory
  - recovery directory
  - LPC asset root
  - whether bundled LPC assets are in use
  - whether `.env` exists locally
  - history persistence mode
  - Gemini mode

### Preference behavior

- Theme preference now applies globally across Studio
- Hiding history tools now removes the Projects route from the main nav on that machine
- Hiding AI assistance now removes the AI guidance panel from the builder on that machine
- Default naming style now seeds a fresh builder workspace when no saved workspace draft exists

## Migration notes

If you are upgrading from 0.36.0:

- no backend data migration is required
- no history schema change is required
- local Studio preferences are stored in browser local storage under a new `studio.preferences.v1` key

## Packaging note

Packaged Windows builds should now include this release as:

```text
build/release/windows/SpriteCraft-win-x64-0.37.0
```
